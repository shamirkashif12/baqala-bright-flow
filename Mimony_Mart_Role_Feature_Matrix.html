Role-Wise Feature Matrix

A complete breakdown of what each of the 9 mobile login roles can see and do — bottom navigation, dashboard, quick actions, and the More menu — derived directly from the app's role-gating logic and the backend's real permission model.

Roles covered

  1. Tenant Administrator (Admin)
  2. Branch Manager (Manager)
  3. Cashier
  4. Supervisor
  5. Inventory Staff (Storekeeper)
  6. Warehouse Manager

Every role also gets four universal items — Policies & Conditions, Settings, Profile, and Sign Out — listed once at the end rather than repeated per role.

1. Tenant Administrator (Admin)

Full system access — every module, every permission flag (View/Create/Edit/Delete/Approve/Export) across all 30 real backend modules. Only role with the 3 ACL screens.
AreaFeatures
Bottom NavDashboard · Orders · POS (center scan button) · Inventory · More
Dashboard
  • Today's Orders / Pending Orders / Low Stock Items / Close-to-Expiry stat cards
  • My Shift card (opening cash, sales, orders) if a shift is open
  • Reports preview: My Shift, Check-In/Out, Opening Cash, Closing Cash (4 tiles, "View all" → Other Reports)
  • Cashier KPI table (today's sales per cashier, mart-wide)
  • Branch Performance table (today's sales per branch)
POS / Orders / Inventory tabs Scan/search products, Cart, Checkout (Cash/Card/Wallet/Split/Transfer), Invoice & receipt printing, Held Orders · Full Orders list with filters · Inventory: view/add/adjust stock, Receive Stock, Wastage & Spoilage entry
More menu
  • Returns — process/approve customer refunds
  • Check-In / Out Report, Opening Cash Report (staff attendance oversight)
  • Other Reports — full 24-report grid (Sales & Performance, Cash & Shifts, Inventory, Terminals & Compliance)
  • Manager Approvals — cash-variance & refund approval center
  • Terminal Bird-Eye — live status of every POS terminal mart-wide
  • Warehouse — stock across every warehouse
  • Pending Transfers — approve/reject stock-transfer requests
  • Audit Logs — personal activity trail
  • Roles & Permissions (ACL) — create/edit roles, 6-flag permission matrix per module
  • User Role Assignment — reassign roles, enable/disable accounts
  • Approval Rules — configure threshold-based approval routing

2. Branch Manager (Manager)

Same operational scope as Tenant Admin, scoped to their own branch server-side; no access to the 3 mart-wide ACL screens.
AreaFeatures
Bottom NavDashboard · Orders · POS (center scan button) · Inventory · More
DashboardIdentical to Tenant Admin's dashboard — order/stock stats, reports preview, Cashier KPI table, Branch Performance table.
POS / Orders / Inventory tabsSame as Tenant Admin — full POS checkout, order management, inventory receive/adjust.
More menu
  • Returns, Check-In/Out Report, Opening Cash Report
  • Other Reports (full 24-report grid)
  • Manager Approvals, Terminal Bird-Eye
  • Warehouse, Pending Transfers
  • Audit Logs (personal activity)
Not shown: Roles & Permissions, User Role Assignment, Approval Rules (Tenant Admin only) · No personal shift check-in/out items (Manager isn't a Cashier — see note below).

3. Cashier

The only role built entirely around running a personal register shift, start to finish.
AreaFeatures
Bottom NavDashboard · Orders · POS (center scan button) · Inventory (view stock only) · More
Dashboard
  • Today's Orders / Pending Orders / Low Stock / Close-to-Expiry stat cards
  • My Shift card — opening float, my sales, my order count
  • Reports preview (My Shift, Check-In/Out, Opening Cash, Closing Cash)
No Cashier-KPI table or Branch-Performance table — manager-only view.
POS tabQuick Scan (Scan for POS Billing / Scan to Add Inventory / Scan Receipt-Order) · Cart · Checkout — Cash, Card, Wallet, Split, Transfer · Invoice/receipt printing · Held Orders
More menu
  • Employee Check-In, Opening Cash, My Shift, Held Orders — cashier-only shift lifecycle
  • Closing Report — close own shift, count cash, submit variance
  • Employee Check-Out
  • Returns — create refunds; over-threshold refunds route to a manager
Not shown: Check-In/Out Report, Opening Cash Report, Other Reports, Manager Approvals, Terminal Bird-Eye, Warehouse, Pending Transfers, Audit Logs, all 3 ACL screens.

4. Supervisor

Manager-tier oversight authority without a POS register — real "POS" module permission is false for this role.
AreaFeatures
Bottom NavDashboard · Orders · Inventory · More (no POS/scan button)
Dashboard
  • Pending Approvals, Today's Sales, Low Stock Items, Branch-name stat cards
  • Quick actions: Manager Approvals, Terminal Bird-Eye, Returns, Cashier Performance Report
More menu
  • Returns, Check-In/Out Report, Opening Cash Report
  • Manager Approvals, Terminal Bird-Eye
  • Warehouse, Pending Transfers
Not shown: Other Reports grid, Audit Logs, POS/Orders-creation, all 3 ACL screens.

5. Inventory Staff (Storekeeper)

Branch-level stock ownership — receiving, adjusting, and returning stock to a warehouse (with approval).
AreaFeatures
Bottom NavDashboard · Inventory · More (no Orders, no POS)
Dashboard
  • Low Stock, Expiring Soon, Pending Returns stat cards
  • Quick actions: Branch Inventory, Receive Stock, Report Waste/Damage, Return to Warehouse (needs Manager/Admin/Warehouse Manager approval), Wastage & Spoilage Report
  • Expiring-batches list (product, batch #, days left)
More menu: no Returns/Reports/Warehouse/Approvals/Audit items — real permissions exclude all of them; everything Inventory Staff needs lives on the dashboard itself.

6. Warehouse Manager (new role)

Full warehouse-operations authority — created this cycle to own supply-to-branch and transfer approvals.
AreaFeatures
Bottom NavDashboard · Warehouse · More
Dashboard
  • Warehouses count, Low Stock, Pending Approvals, Stock Value stat cards
  • Quick actions: Warehouses, Supply to Branch (create warehouse→branch transfer), Pending Transfers
  • Recent Transfers list (source → destination, status, item count)
More menuPending Transfers
Warehouse item omitted from More menu — already the dedicated bottom-nav tab. No Returns/Other-Reports/Audit access.

7. Warehouse Staff

Day-to-day warehouse floor work — view-only on transfers; approving/receiving is a Warehouse Manager action.
AreaFeatures
Bottom NavDashboard · Warehouse · More
Dashboard
  • Warehouses, Items in Stock, Low Stock, Incoming Shipments stat cards
  • Quick action: Warehouse Stock (view levels + item detail)
  • Incoming Shipments list — read only, no receive/approve button (real permission has no Approve flag on Stock Transfers for this role)
More menu— (universal items only)

8. Auditor

Pure read-only oversight — no Create/Edit/Approve flags on any module in the real permission matrix.
AreaFeatures
Bottom NavDashboard · More (no Orders, POS, Inventory, or Warehouse tab)
Dashboard
  • Critical Events, Failed Logins, Overrides, Configuration Changes stat cards (rolling 7-day Audit Trail KPIs)
  • Quick actions: Audit Trail, VAT/ZATCA Report, Discounts Report, My Activity (personal sign-in/action log)
  • Recent Events list (action, user, module, severity)
More menu— (universal items only)
Deliberately excluded everywhere: Returns, Warehouse, Other-Reports grid, Manager Approvals, Pending Transfers — an Auditor's oversight tools are all on the dashboard, and none of them are edit-capable.

9. Accountant (Finance User)

Finance/tax oversight plus the one real approval action this role owns: expense sign-off.
AreaFeatures
Bottom NavDashboard · More
Dashboard
  • Pending Expenses, VAT Collected, Gross Margin, Gross Profit stat cards
  • Quick actions: VAT/ZATCA Report, Tax Report, Fees Report, Profit Margin Report
  • Pending Expenses list with real Approve/Reject buttons
More menuReturns (view/approve), Warehouse (view)
Not shown: Check-In/Out or Opening Cash reports, Other Reports grid, Manager Approvals, Terminal Bird-Eye, Audit Logs, Pending Transfers, ACL screens.
Universal, every role: Policies & Conditions · Settings · Profile · Sign Out.

Not covered: Self-Checkout Kiosk exists as a real backend role but drives the separate self-checkout web kiosk product, not a staff mobile login, so it's excluded here.
